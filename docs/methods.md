# Methods

## Study Design and Participants

We analyzed simultaneous scalp EEG and stereo-electroencephalography (SEEG) recordings from 20 patients undergoing invasive monitoring for presurgical epilepsy evaluation. The primary analysis used interictal resting-state segments from the IDE-A condition, which served as the reference state for all main EEG-SEEG coupling and SEEG archetype analyses. Seizure-stage analyses were treated as a clinical extension and projected preictal, low-voltage fast activity (LVFA), ictal, and postictal segments into the fixed IDE-A-derived state space.

The final manuscript should report the institutional review board approval identifier, consent procedure, medication status when available, seizure diagnosis, implantation indication, and exclusion criteria. No analysis step used patient-identifying labels in manuscript-facing outputs; subject identifiers were converted to stable anonymized aliases for figure generation and audit tables.

## Data Organization and Cohort Indexing

The workflow expected one folder per patient containing reference EEG recordings, bipolar SEEG recordings, and an electrode atlas table (`Atlas.tsv`). Segment timing and seizure-stage annotations were read from the study workbook. Eligible IDE-A records were indexed only when the required EEG, bipolar SEEG, atlas, and timing metadata were available. For seizure-stage analyses, the workflow separately indexed preictal, LVFA, ictal, and postictal segments from workbook-derived seizure annotations and recorded stage timing, processing eligibility, paired EEG-SEEG availability, and quality-control status.

All analysis artifacts were generated by a cache-aware Python workflow and exported as audit-ready tables before figure rendering. Python code under `src/seeg_eegmicrostates/` performed indexing, preprocessing, state labeling, coupling analysis, statistics, and table export. R Markdown scripts under `scripts/` consumed exported CSV/XLSX tables or cached summaries to render manuscript-ready figures, without re-estimating the core analyses.

## EEG Preprocessing and Microstate Labeling

Scalp EEG recordings were cropped to the selected analysis segment, band-pass filtered from 1 to 40 Hz, resampled to 250 Hz, projected to a standard 19-channel 10-20 montage, and average referenced. The workflow first normalized the available scalp EEG to a shared 11-channel layout (`F3`, `Fz`, `F4`, `C3`, `Cz`, `C4`, `P3`, `Pz`, `P4`, `O1`, `O2`) and then expanded it to the target 19-channel layout (`Fp1`, `Fp2`, `F7`, `F3`, `Fz`, `F4`, `F8`, `T7`, `C3`, `Cz`, `C4`, `T8`, `P7`, `P3`, `Pz`, `P4`, `P8`, `O1`, `O2`) by marking absent channels as bad and interpolating them after montage assignment. This ensured a common scalp topography space across patients.

EEG microstates were labeled using a `pycrostates` modified k-means (`ModKMeans`) template solution with `K = 4`, consistent with the classical EEG microstate literature. The active template was loaded from the configured FIF template file unless an explicit template override was provided. Template compatibility was checked against either the restored 19-channel layout or the shared 11-channel layout. Continuous EEG was assigned to the best-fitting template with polarity-invariant spatial correlation and a minimum segment duration of 30 ms. For each sample, the workflow retained the assigned microstate label and the corresponding absolute spatial-correlation score. EEG global field power (GFP) was computed as the across-channel standard deviation, and GFP peak tables were generated using a minimum peak distance of 10 ms.

## SEEG Preprocessing and Bipolar Channel Mapping

Bipolar SEEG recordings were cropped to the same analysis segment, band-pass filtered from 1 to 40 Hz, and resampled to 250 Hz. Bipolar channels were mapped to anatomical or functional labels by parsing the two contact names in each bipolar pair and retaining pairs for which both contacts belonged to the same atlas-defined unit. The main manuscript-facing common space used the Schaefer/Yeo 17-network annotation column (`cortex_319663V:Schaefer_200_17net`) to summarize electrode coverage and to project field-state templates into a cross-patient network space. AAL3 mapping remained available for historical or supplementary analyses, but the retained main-text archetype workflow used Yeo17 for the group comparison space.

For each patient, SEEG channel values were normalized within channel before field-state extraction. The retained main analysis used conventional z-scoring; a robust median absolute deviation z-score option was available for sensitivity analyses. Channels with non-finite values or zero variance were excluded from state estimation.

## Patient-Level SEEG Global-Field State Extraction

SEEG global-field states were estimated independently within each patient. At each time sample, the normalized bipolar-channel vector was centered and scaled to unit norm, yielding polarity-invariant spatial patterns. A SEEG global-field magnitude trace was then computed from the normalized bipolar signals. The retained peak metric was the root mean square (RMS) across bipolar channels; a spatial-standard-deviation metric was available for sensitivity analyses. Local maxima of this trace were used as candidate SEEG peak maps, with a minimum peak distance of 10 ms and an upper sampling cap of 5000 peak maps per patient when required.

Peak maps were clustered using a custom polarity-invariant k-means procedure. Templates were initialized from random peak maps, peak maps were assigned by absolute spatial similarity, member maps were sign-aligned to their current template, and templates were updated by the normalized mean pattern. Clustering was iterated until labels and templates stabilized or until the maximum iteration count was reached. Cluster labels were then backfit to the continuous SEEG time series by assigning each normalized sample to the most similar template. Runs shorter than 30 ms were smoothed by merging them into an adjacent state chosen by local template-fit evidence.

For each patient and field state, the workflow exported the template pattern, occupancy, mean dwell time, number of samples, number of runs, peak-map support, and transition probabilities. Occupancy was defined as the fraction of segment samples assigned to a state. Dwell time was computed from contiguous state runs and expressed in seconds or milliseconds depending on the exported table.

## Model-Order Evaluation

The main analysis retained `K = 4` SEEG field states per patient. To avoid relying solely on convention, the workflow evaluated candidate model orders from `K = 2` to `K = 7`. For each candidate `K`, patient-level summaries included mean, median, and minimum peak-map template fit; split-half template stability; minimum and maximum state occupancy; peak-map support fractions; and occupancy entropy. Split-half stability was estimated by dividing each patient's peak maps into two random halves, fitting templates independently in each half, and computing the best unique polarity-invariant template-set similarity between halves.

Group-level model-order summaries aggregated these metrics across patients and marked the retained main-text default. The retained `K = 4` solution was used because it balanced template fit, diminishing incremental gain, split-half stability, interpretability, and continuity with the four-class EEG microstate framework.

## Projection to a Common Yeo17 Archetype Space

Because SEEG electrode coverage differs across patients, patient-level field-state templates were not compared directly in native electrode space. Instead, each template was projected to a common Yeo17 network representation. For each patient and field-state template, valid same-network bipolar channels were grouped by Yeo17 network, and the mean template loading was computed within each network. The resulting network vector was centered, normalized, and sign-oriented according to its dominant absolute loading so that polarity did not drive cross-patient matching.

Group SEEG archetypes were then derived by clustering the patient-level projected templates in Yeo17 space with the same polarity-invariant template procedure. Each patient's field states were matched to group archetypes by maximizing absolute similarity under a unique-assignment constraint when feasible. For every assignment, the workflow stored the assigned archetype, assignment similarity, orientation sign, and support metadata. Archetype support was summarized as the number of patients contributing to each group archetype and the distribution of assignment similarities.

To evaluate whether archetype assignments exceeded chance label matching, a permutation-based null reference was generated by enumerating alternative patient-level field-state-to-archetype assignments and recomputing assignment similarities. Observed assignment similarities were then compared against this null distribution in the manuscript-facing validation figures.

## EEG-SEEG Temporal Alignment and Fine-Lag Coupling

EEG microstate labels, EEG GFP traces, SEEG field-state labels, and SEEG archetype labels were aligned on the shared 250 Hz analysis grid using nearest-neighbor time matching with a 5 ms tolerance. The principal cross-modal synchrony analysis quantified whether EEG microstate labels and SEEG field-state or archetype labels co-occurred more often than expected under temporal-shift null models.

For each patient, binary co-occurrence was evaluated at lags from -40 ms to +40 ms in 4 ms steps, corresponding to the native 250 Hz sample spacing. Positive lags indicated that the SEEG state series was shifted forward relative to EEG labels; negative lags indicated the opposite direction. At each lag, observed coupling was compared with circular-shift surrogate couplings generated within patient. The retained SEEG field-state surrogate count was 128 unless explicitly overridden in the command-line workflow. Subject-level effect size was defined as observed coupling minus the mean surrogate coupling. Subject peak summaries included the lag of the maximal effect, peak effect size, observed and null coupling at the peak, permutation p value, and peak width.

Group-level inference used patient-first summaries. For each lag, subject-level effects were tested against zero using sign-flip permutation testing, and p values were corrected across tested lag points using the Benjamini-Hochberg false-discovery-rate procedure. The same patient-first framework was used for state preference, GFP/global coupling, transition, and switching analyses where applicable.

## Archetype-Conditioned EEG Topography

To interpret how SEEG archetypes related to scalp EEG spatial patterns, instantaneous EEG topographies were aligned with SEEG archetype labels. EEG topographies were centered, normalized, and sign-oriented before averaging. For each patient and SEEG archetype, the workflow averaged all aligned EEG topographies assigned to that archetype and exported the conditioned scalp map together with sample support and the number of represented EEG microstates. Group-level conditioned EEG maps were computed by averaging patient-level conditioned maps for each archetype when the minimum subject-support threshold was met.

Conditioned EEG maps were compared with the four EEG microstate templates using polarity-invariant spatial similarity. In parallel, the workflow estimated conditional state preference, defined as the difference between `P(EEG microstate | SEEG archetype)` and the patient-specific baseline EEG microstate probability. Subject-level preference effects were aggregated across patients with sign-flip permutation testing and false-discovery-rate correction.

Representative single-subject conditioned EEG panels were selected deterministically from exported support metrics. Preference was given to patients with complete archetype panels, higher minimum conditioned sample support, higher mean assignment similarity, and larger total conditioned sample counts. Displayed subject labels were anonymized.

## GFP-Informed Global Coupling and Switching Analyses

Supplementary analyses assessed whether EEG GFP and SEEG global magnitude shared near-zero-lag coupling and whether SEEG state transitions predicted EEG microstate switching. Yeo17 network-level SEEG signals were combined into a global magnitude trace using the retained global-metric branch. Lagged EEG GFP-SEEG global coupling was estimated with circular-shift surrogate baselines and patient-first group statistics. Peak-centered analyses summarized SEEG global trajectories around EEG GFP peaks.

Switching analyses tested whether transitions between SEEG field states or archetypes were followed by EEG microstate switching within a 250 ms window. For each transition class, observed response rates were compared with circularly shifted EEG or SEEG label surrogates. GFP-controlled switching models additionally regressed switching indicators on centered EEG GFP and a post-transition indicator to estimate transition-related effects after accounting for GFP variation.

## Seizure-Stage Projection and IDE-A Baseline Comparison

Seizure-stage analyses were designed as a projection into the fixed IDE-A reference state space rather than as a new state-discovery analysis. For each seizure-stage segment, EEG was labeled using the retained IDE-A EEG microstate template, and SEEG was projected into each patient's IDE-A field-state templates and group archetype assignments. This preserved a common state vocabulary across IDE-A, preictal, LVFA, ictal, and postictal periods.

Stage-level outputs included EEG microstate metrics, SEEG field-state metrics, SEEG archetype metrics, transition metrics, Yeo17 loading summaries, and paired EEG-SEEG relationship tables. Primary clinical comparisons used patient-first matched deltas:

```text
delta(stage) = patient_mean(stage) - patient_mean(IDE-A)
```

Repeated seizures within the same patient and stage were averaged before group summaries. Occupancy and dwell time were treated as the primary state-dynamic metrics. Yeo17 loading changes were interpreted as secondary spatial-fingerprint summaries, and EEG-SEEG conditional probability changes were treated as tertiary exploratory relationship metrics. The R Markdown seizure-stage reports separated stage-only descriptive trajectories from IDE-A baseline comparisons, and exported denominator, duration, projection-confidence, complete-case, seizure-type, and repeated-seizure sensitivity tables.

## Statistical Analysis

All inferential summaries followed a patient-first strategy. Subject-level quantities were first computed within patient, and group-level inference was performed across patient-level effects to avoid treating time samples, channels, or seizures as independent observations. One-sample or paired effects were tested using sign-flip permutation tests with 2048 permutations unless otherwise specified. Repeated-measures profile effects used within-subject label permutations and a repeated-measures F statistic. Pairwise post hoc contrasts used patient-level paired differences and sign-flip permutation tests.

Permutation p values used a plus-one correction:

```text
p = (number of null statistics at least as extreme as observed + 1) / (number of permutations + 1)
```

Multiple comparisons were controlled with the Benjamini-Hochberg false-discovery-rate procedure within each analysis family. Reported group summaries included the number of contributing subjects, mean effect, median effect, permutation p value, and FDR-adjusted q value. Descriptive panels additionally reported medians, interquartile ranges, bootstrap confidence intervals for selected paired contrasts, and support counts where these were more informative than formal testing.

## Figure Generation and Reproducibility

All manuscript-facing figures were rendered from exported CSV/XLSX tables or stable cached artifacts. The main validation figure workflow exported Yeo17 cohort coverage, field-state transition summaries, model-order diagnostics, archetype support and assignment-null references, lag-decomposition inputs, and conditioned EEG topography tables. R Markdown scripts rendered SVG outputs for cohort coverage, field-state reproducibility, lag decomposition, group conditioned EEG topographies, and representative single-subject examples. Separate R Markdown workflows rendered Yeo17 archetype brain maps, archetype parameter summaries, EEG-SEEG relationship panels, EEG microstate overviews, seizure-stage trajectories, and IDE-A versus seizure-stage comparison panels.

This separation ensured that statistical outputs could be audited independently of manuscript styling. Each figure family wrote both the plotted SVG files and the CSV side tables used to generate them.

## Software

Analyses were implemented in Python using MNE-Python for EEG/SEEG data handling and preprocessing, pycrostates for EEG microstate model handling and labeling, NumPy and pandas for numerical and tabular computation, and SciPy for peak detection and interpolation. Manuscript figures were rendered in R using R Markdown, ggplot2, patchwork, ggdist, ggtext, paletteer, svglite, ggseg, and ggsegYeo2011. The project used a `src`-layout Python package with reproducible dependency management through `uv`; generated analysis outputs were cached under hashed runtime branches.
