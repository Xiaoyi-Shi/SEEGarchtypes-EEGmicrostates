## 1. 当前职责边界

主工程 `src/seeg_eegmicrostates/` 负责：

- 建立 IDE 受试者索引
- 生成 EEG microstate 标签、模板投影和 GFP 结果
- 生成 SEEG parcellation 信号
- 运行论文主线 exploratory 分析
- 构建 seizure-stage 索引与阶段投影分析
- 导出 manuscript-ready 表格、manifest 和 R 可读 CSV

`scripts/` 负责：

- R Markdown 图像渲染
- 论文辅助导出脚本
- 手工排版前的 SVG / CSV 整理
- 其他一次性检查工具

原则上：

- `src/` 不直接负责论文图像排版
- `scripts/` 不重算主分析，只消费缓存、导表或少量可审计辅助导出

## 2. 论文主线 CLI 流程

推荐主线分析空间使用 `Yeo17`。标准顺序如下：

```powershell
uv run seeg-eegmicrostates build-index --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-eeg-states --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-seeg-regions --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-exploratory-coupling --analysis all --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates export-paper-tables --seeg-parcellation-name yeo17
```

如果希望多个步骤共用同一个运行目录，统一加 `--run-id`：

```powershell
uv run seeg-eegmicrostates build-index --run-id 20260428_main --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-eeg-states --run-id 20260428_main --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-seeg-regions --run-id 20260428_main --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-exploratory-coupling --analysis all --run-id 20260428_main --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates export-paper-tables --run-id 20260428_main --seeg-parcellation-name yeo17
```

说明：

- `run-exploratory-coupling` 的默认主线已经围绕 field-state / archetype / lag / GFP / switching 组织
- `field-archetype-space` 主文默认使用 `yeo17`
- `export-paper-tables` 只导表，不在 Python 里直接画主文图

## 4. 癫痫阶段扩展流程

如果要做 `pre / LVFA / SZ / post` 分析，使用 seizure-stage 分支：

```powershell
uv run seeg-eegmicrostates build-seizure-stage-index --run-id seizure_stage_main --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates run-seizure-stage-analysis --run-id seizure_stage_main --seeg-parcellation-name yeo17
uv run seeg-eegmicrostates export-seizure-stage-tables --run-id seizure_stage_main --seeg-parcellation-name yeo17
```

这条分支的作用是：

- 从 `datas/info_patient.xlsx` 读取手工标注的发作阶段
- 生成阶段 timing / QC / paired eligibility
- 将各阶段投影到固定的 IDE-A EEG microstate 与 SEEG archetype 参考空间
- 导出 `07/08` R Markdown 所需的 CSV

## 6. 图像与手工工作流

### 6.1 Manifest-driven 主图与补图

导表后可以直接渲染：

```powershell
Rscript -e "rmarkdown::render('scripts/01_main_figures.Rmd', params=list(report_root='artifacts/runs/<run-id>/reports'))"
Rscript -e "rmarkdown::render('scripts/02_supplementary_figures.Rmd', params=list(report_root='artifacts/runs/<run-id>/reports'))"
```

这两份文档读取 `reports/manifests/` 下的 manifest，并把标准化主图/补图输出到 `reports/main_figures/` 与 `reports/supplementary_figures/`。

### 6.2 手工主文图工作流

当前保留的手工图像脚本主要是：

- `scripts/03_patient_archetype_yeo17_ggseg_maps.Rmd`
  绘制 group archetype 的 `Yeo17` 脑图。
- `scripts/04_patient_archetype_parameter_statistics.Rmd`
  绘制 archetype 的 occupancy、dwell time、assignment similarity、Yeo17 loading 等统计图。
- `scripts/05_seeg_archetype_eeg_microstate_relationship.Rmd`
  绘制 SEEG archetype 与 EEG microstate 的关系图。
- `scripts/export_eeg_microstate_overview.py` + `scripts/06_eeg_microstate_overview.Rmd`
  绘制 EEG microstate 单独总览。
- `scripts/07_seizure_stage_microstate_archetype_trajectory.Rmd`
  阶段内描述性轨迹图，不直接和 IDE-A baseline 做差。
- `scripts/08_ide_a_vs_seizure_stage_comparison.Rmd`
  IDE-A baseline vs seizure-stage comparison 主分析图。

这些脚本的详细输入输出见 [scripts/README.md](</F:/uv_env/SEEG-EEGmicrostates/scripts/README.md>)。

## 8. 注意事项

- CLI 默认 SEEG parcellation 仍然是 `aal3`，主文分析时应显式传 `--seeg-parcellation-name yeo17`
- `Figure 1C` 现在也改为 `Yeo17` coverage，不再使用旧的 `Yeo7` 版本
- `scripts/09_maintext_validation_figures.Rmd` 渲染时会在 `scripts/` 下生成 `.html` 和 `.knit.md` 中间文件；它们不是正式源码，不需要作为分析结果引用
- 如果只想刷新手工图而不重跑主分析，优先复用 `artifacts/cache/`、`artifacts/runs/.../reports/` 和 `artifacts/manual/` 中已经存在的中间产物
