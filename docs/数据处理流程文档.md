# SEEG-EEG Microstates 数据处理流程文档

## 1. 文档目的

这份文档描述当前仓库中已经实现并正在使用的完整分析流程。它对应的是公开 CLI 工作流，而不是历史遗留分支。

当前默认分析路径有三个核心约束：

- EEG 使用 `1-40 Hz` 预处理后的信号，并默认加载配置好的模板 `artifacts/cache/eeg/ModK.fif`
- SEEG 使用 `AAL3` 解剖分区，并且只保留“双极道两端都落在同一个 AAL3 区域”的通道
- 主流程默认分析 `IDE_A`，但 `IDE_S` 也是公开支持的可选工作流
- 如果进入 GFP-informed exploratory，全局耦合分支当前要求 `Yeo17` 网络分区
- 如果进入 SEEG global-field-state exploratory，状态模板来自病人内双极道 peak maps，默认不假设跨病人 state label 一一对应
- 如果进入 SEEG global-field-state archetype exploratory，跨病人比较默认先投影到 `Yeo17` 共同空间，再做 sign orientation 和 archetype matching
- 可复用缓存固定写入 `artifacts/cache/`，每次 CLI 命令调用的结果和日志写入 `artifacts/runs/<run-id>/`

主流程包含：

1. `build-index`
2. `run-eeg-states`
3. `run-seeg-regions`
4. `run-activity-effects`
5. `run-connectivity-effects`
6. `render-reports`

可选的探索性分析通过 `run-exploratory-coupling` 单独触发，不改变默认主流程的缓存契约。

补充说明：

- 如果不显式传 `--run-id`，CLI 会自动使用当前时间戳作为运行目录名
- 如果多步命令传入相同的 `--run-id`，它们会把日志和报告归并到同一个 `artifacts/runs/<run-id>/` 目录

## 2. 输入数据

默认输入位置：

- `datas/data_01_seeg/<patient>/ref/*.fif`
- `datas/data_01_seeg/<patient>/bipolar/*.fif`
- `datas/data_01_seeg/<patient>/MNI/Atlas.tsv`
- `datas/info_patient.xlsx`

各类输入的作用：

- `ref/*.fif`：用于 EEG 预处理和 microstate 标注
- `bipolar/*.fif`：用于 SEEG 双极信号分析
- `Atlas.tsv`：提供每个接触点的 `AAL3` 分区标签
- `info_patient.xlsx`：提供病人信息和所选 IDE 状态时间段注释，默认是 `IDE_A`

## 3. 全局默认参数

当前实现中的关键默认值如下：

- EEG 默认模板：`artifacts/cache/eeg/ModK.fif`
- 默认分析状态：`IDE_A`
- 可选分析状态：`IDE_S`
- EEG 公共工作流带通：`1-40 Hz`
- SEEG 带通：`1-40 Hz`
- EEG 重采样率：`250 Hz`
- SEEG 重采样率：`250 Hz`
- SEEG 默认幅度缩放：`raw`，不做通道级或区域级 `z-score`
- microstate 数量：`4`
- 最小 microstate 时长：`30 ms`
- exploratory 默认事件窗：`1.0 s`
- exploratory 默认滑窗长度：`10.0 s`
- 组水平最小受试者支持阈值：`7`

说明：

- 如果默认模板文件不存在，`run-eeg-states` 会直接报错，不会自动回退成重新拟合模板

## 5. 主流程分步说明

### 5.1 `build-index`

命令：

```bash
uv run seeg-eegmicrostates build-index
```

这个阶段完成四件事：

1. 读取 `datas/info_patient.xlsx`
2. 扫描 `datas/data_01_seeg/` 下的 EEG、SEEG、Atlas 文件
3. 从 workbook 中提取所选分析状态的时间段，默认是 `IDE_A`
4. 生成主分析队列 `include_main`

主要输出：

- `artifacts/cache/index/recording_index_*.parquet`
- `artifacts/cache/segments/ide_a_segments_*.parquet`
- `artifacts/cache/index/cohort_ide_a_main_*.parquet`
- `artifacts/cache/eeg/eeg_channel_inventory_*.parquet`

这一阶段的结果决定后面哪些病人会进入公开主流程。

### 5.2 `run-eeg-states`

命令：

```bash
uv run seeg-eegmicrostates run-eeg-states
```

可选覆盖：

```bash
uv run seeg-eegmicrostates run-eeg-states --template-fif path/to/other_template.fif
```

每个病人的 EEG 处理步骤如下：

1. 从 `ref/*.fif` 读取数据
2. 按所选分析状态起止时间裁剪，默认是 `IDE_A`
3. 统一 EEG 通道命名
4. 只保留共享 `11` 导联：
   `F3/Fz/F4/C3/Cz/C4/P3/Pz/P4/O1/O2`
5. 带通滤波到 `1-40 Hz`
6. 重采样到 `250 Hz`
7. 投影到标准 `19` 导联布局：
   `Fp1,Fp2,F7,F3,Fz,F4,F8,T7,C3,Cz,C4,T8,P7,P3,Pz,P4,P8,O1,O2`
8. 对缺失导联做插值恢复
9. 应用平均参考
10. 优先加载 `--template-fif` 指定模板；否则加载配置默认模板 `artifacts/cache/eeg/ModK.fif`
11. 用该模板对每个时间点打 EEG microstate 标签，并输出匹配置信度 `corr`

这一阶段当前不是重新拟合模板，而是“配置模板/自定义模板 + 重标注”。写出的 `group_microstate_model_band_1_40_*.fif` 是当前生效模板在缓存中的副本，便于后续报告渲染和结果追踪。如果默认模板不存在且没有传 `--template-fif`，流程会直接报错。

主要输出：

- `artifacts/cache/eeg/group_microstate_model_band_1_40_*.fif`
- `artifacts/cache/eeg/microstate_labels_band_1_40_*.parquet`
- `artifacts/cache/eeg/gfp_trace_band_1_40_*.parquet`
- `artifacts/cache/eeg/gfp_peaks_band_1_40_*.parquet`
- `artifacts/cache/eeg/restored_channels_band_1_40_*.parquet`
- `artifacts/cache/eeg/<patient>_preprocessed_band_1_40_*.fif`

标签表的核心列：

- `patient_id`
- `time_sec`
- `sample`
- `microstate`
- `corr`

### 5.3 `run-seeg-regions`

命令：

```bash
uv run seeg-eegmicrostates run-seeg-regions
```

每个病人的 SEEG 处理步骤如下：

1. 从 `bipolar/*.fif` 读取双极信号
2. 按所选分析状态起止时间裁剪，默认是 `IDE_A`
3. 读取该病人的 `Atlas.tsv`
4. 解析每个双极通道的两端接触点
5. 读取接触点在 `AAL3` 列中的标签
6. 仅保留“两端接触点标签都存在，且二者完全相同”的双极道
7. 对双极信号带通到 `1-40 Hz`
8. 重采样到 `250 Hz`
9. 对同一 `AAL3` 区域内的所有有效双极道取均值，形成区域时序
10. 当前默认流程直接保留通道和区域的原始相对幅度，不做通道级或区域级 `z-score`

核心思想是严格的 `same-region bipolar only`：

- 保留逻辑最干净
- 避免跨区双极道归属不清
- 代价是部分接触点被丢弃

这样做的含义是：

- 区域均值会保留原始带限信号的相对幅度关系
- downstream activity / connectivity 会更直接反映真实幅度差，而不是标准化后的单位方差信号

主要输出：

- `artifacts/cache/seeg/bipolar_region_map_band_1_40_*.parquet`
- `artifacts/cache/seeg/region_coverage_band_1_40_*.parquet`
- `artifacts/cache/seeg/<patient>_region_band_limited_band_1_40_*.parquet`

映射表的核心列：

- `bipolar_channel`
- `contact_a`
- `contact_b`
- `region_a`
- `region_b`
- `region`
- `valid_same_region`

覆盖表的核心列：

- `patient_id`
- `region`
- `n_bipolar_channels`

### 5.4 `run-activity-effects`

命令：

```bash
uv run seeg-eegmicrostates run-activity-effects
```

这个阶段做的是 EEG 状态条件下的 SEEG 区域活动比较。

处理步骤：

1. 读取 EEG microstate 标签
2. 读取每个病人的 AAL3 区域时序
3. 通过时间戳近邻对齐 EEG 标签与 SEEG 区域时序
4. 将宽表区域时序转成长表
5. 对每个病人的每个 `microstate x region` 计算四状态 profile：
   - `state_mean`
   - `state_std`
   - `n_state_samples`
6. 在组水平对每个 region 施加最小受试者阈值，默认 `7`
7. 对每个 region 做 four-state omnibus 检验
8. 对每个 region 的四个状态做 pairwise post-hoc 比较
9. 对 omnibus 和 post-hoc 结果分别做 FDR 校正

主要输出：

- `artifacts/cache/coupling/aligned_activity_*.parquet`
- `artifacts/cache/coupling/subject_activity_profiles_*.parquet`
- `artifacts/cache/stats/group_activity_omnibus_*.parquet`
- `artifacts/cache/stats/group_activity_posthoc_*.parquet`

### 5.5 `run-connectivity-effects`

命令：

```bash
uv run seeg-eegmicrostates run-connectivity-effects --method all
```

支持三种连接指标：

- `corr`
- `plv`
- `wpli`

处理步骤：

1. 读取 EEG microstate 标签
2. 读取 AAL3 区域时序
3. 对齐 EEG 标签和 SEEG 区域时序
4. 对每个病人的每个 `microstate x region_a x region_b x method` 计算 state-specific connectivity profile：
   - `state_connectivity`
   - `n_state_samples`
5. 对每个区域对做 four-state omnibus 检验
6. 对每个区域对的四个状态做 pairwise post-hoc 比较
7. 对组水平结果施加最小受试者阈值，默认 `7`
8. 对 omnibus 和 post-hoc 结果分别做 FDR 校正生成 group-level 表

三种连接指标的实现方式：

- `corr`：绝对 Pearson 相关
- `plv`：Hilbert 相位后的 phase locking value
- `wpli`：Hilbert 相位后的 weighted phase lag index

主要输出：

- `artifacts/cache/coupling/aligned_connectivity_band_1_40_<method>_*.parquet`
- `artifacts/cache/coupling/subject_connectivity_profiles_band_1_40_<method>_*.parquet`
- `artifacts/cache/stats/group_connectivity_omnibus_band_1_40_<method>_*.parquet`
- `artifacts/cache/stats/group_connectivity_posthoc_band_1_40_<method>_*.parquet`

### 5.6 `render-reports`

命令：

```bash
uv run seeg-eegmicrostates render-reports
```

这个阶段不重新计算分析，而是从现有缓存中生成图和 Excel 表。

默认会渲染：

- AAL3 覆盖图
- EEG microstate 模板图
- activity omnibus heatmap
- activity post-hoc heatmap
- connectivity omnibus 矩阵图
- connectivity post-hoc 矩阵图
- 对应的 subject-level profile / group-level omnibus / group-level post-hoc Excel 表

如果 exploratory 缓存存在，还会自动发现并渲染 exploratory 图表，无需单独指定。

需要注意的是，`render-reports` 会把图和 Excel 输出写到当前 `run-id` 对应的目录下。如果每一步都使用默认自动时间戳，那么从零开始依次运行整条管线时，通常会得到多个并列的运行目录；如果所有步骤复用同一个 `--run-id`，这些日志和报告就会落在同一个目录里。


在所有 exploratory 分析之前，系统会先从 EEG 标签中构建共享事件表：

- `exploratory_event_table`
- `exploratory_transition_table`

这些表来自 EEG microstate 的连续片段分解。

### 6.1 Event-triggered activity

问题：

- 当 EEG 进入某个 microstate 时，SEEG 区域活动是否发生系统性变化

步骤：

1. 从 EEG 标签中提取 onset 事件
2. 对每个事件切 `[-window, 0)` 与 `[0, +window)` 两段
3. 计算每个区域的 `post - pre`
4. 先在病人内做事件平均
5. 再做 group-level permutation + FDR

默认参数：

- `event_window_sec = 1.0`

### 6.2 Event-triggered connectivity

问题：

- EEG 状态事件前后，区域间连接是否重组

步骤：

1. 使用 EEG onset 事件
2. 对事件前后窗口分别计算区域对连接
3. 在每个病人内平均事件效应
4. 进入 group-level permutation + FDR

支持：

- `corr`
- `plv`
- `wpli`

### 6.3 Windowed coupling

问题：

- 在较慢时间尺度上，EEG 状态占比和 SEEG 区域活动是否协同变化

步骤：

1. 对 EEG 标签构建滑窗指标：
   - `occupancy`
   - `mean_dwell_sec`
2. 对 SEEG 区域时序构建同窗指标：
   - `mean_value`
   - `std_value`
3. 合并同一时间窗内的 EEG 与 SEEG 指标
4. 在病人内计算：
   - `slope`
   - `effect_mean_diff`，这里实际是 occupancy 与区域值的相关系数
5. 进入 group-level permutation + FDR

默认参数：

- `window_sec = 10.0`

### 6.4 Transition coupling

问题：

- EEG 从一个状态转向另一个状态时，SEEG 区域活动是否有转移特异性变化

步骤：

1. 从 EEG 标签连续片段中提取转移事件
2. 记录 `from_state -> to_state`
3. 在转移点前后取事件窗
4. 对每个病人、每个 `from_state x to_state x region` 计算 `post - pre`
5. 进入 group-level permutation + FDR

默认参数：

- `event_window_sec = 1.0`

### 6.5 Direct state coupling

问题：

- EEG microstate 序列和 SEEG 降维状态序列在同一时间轴上是否存在直接耦合

步骤：

1. 从已缓存的 SEEG 区域时序构建 reduced-space 表示，当前默认 backend 是 `pca-kmeans`
2. 在 reduced-space 上推导 SEEG state labels
3. 将 EEG microstate labels 与 SEEG state labels 按时间戳对齐
4. 计算零滞后 direct coupling 指标
5. 通过 circular-shift surrogate 生成保持时间结构的 null 分布
6. 先在病人内形成效应量，再做 group-level sign-flip permutation + FDR

### 6.6 Lagged state coupling

问题：

- EEG state 与 SEEG state 的直接耦合是否出现在非零滞后上

步骤：

1. 复用 direct state coupling 的 EEG/SEEG state labels
2. 在 `[-max_lag_ms, +max_lag_ms]` 范围内按 `lag_step_ms` 扫描滞后
3. 对每个 lag 计算 patient-level direct coupling 指标
4. 通过 circular-shift surrogate 生成结构化 null 分布
5. 在组水平汇总每个 lag 的效应

### 6.7 Transition state coupling

问题：

- 当 EEG 发生 `from_state -> to_state` 转移时，SEEG state transition 是否更容易在给定时间窗内出现

步骤：

1. 复用 EEG transition table
2. 从 SEEG state labels 中生成 SEEG transition table
3. 对每个 EEG 转移事件检查后续 `transition_window_sec` 时间窗内是否出现 SEEG transition
4. 得到每个病人的 `from_state x to_state` 转移匹配率
5. 通过 circular-shift surrogate 评估 null，并在组水平汇总

### 6.8 SEEG global-field-state coupling

问题：

- SEEG 是否也存在类似 EEG GFP peaks 的病人内稳定空间构型，并且这些 field states 是否和 EEG microstates 有同步、滞后或 transition 关系

当前实现的默认路径：

1. 直接从裁剪后的双极道 SEEG 开始，而不是从 AAL3/Yeo17 均值开始
2. 先做病人内每个双极道 `z-score`
3. 用跨双极道 `RMS` 定义主 peak metric，并在 peak 时刻提取瞬时 bipolar map
4. 用极性不变相似度 `|corr|` 聚类得到病人内 SEEG field templates
5. 对连续时间序列做 backfitting，生成连续 `field_state` labels
6. 输出同步、lagged、transition-conditioned，以及 GFP-controlled switching follow-up

解释限制：

- 这些 `field_state` 模板当前是病人内定义的
- 默认不把某个病人的 `state 0` 解释成另一个病人的同名状态

这一层之上新增了三类 follow-up：

- `field-state-archetypes`
  把病人内模板先投影到共同空间，默认是 `Yeo17`
  再做 sign orientation 和 subject-to-archetype matching
  输出 group archetype templates、assignment similarity 和 support diagnostics
- `archetype-conditioned-eeg-topography`
  把已经匹配到群体 archetype 的 SEEG field-state 标签对齐到 staged EEG 传感器空间样本
  先输出 patient-level 和 group-level archetype-conditioned EEG scalp maps
  再与 staged EEG microstate templates 做空间相似性比较，并保留零时滞偏好或分布式对应关系
- `fine-lag-field-state-coupling`
  只围绕近零时滞扫描，不再用粗 `100 ms` 网格
  而是在共享 `250 Hz` 时间轴上按原生 `4 ms` 步长评估同步峰
- `field-state-to-eeg-switching`
  把主要 mechanistic follow-up 改成 `SEEG -> EEG`
  优先问 SEEG field-state transition 后 EEG microstate 是否更容易切换

原来的 `transition-field-state-coupling` 和 `gfp-controlled-field-state-switching` 仍保留，但更适合作为次级诊断，而不是主生理解释。

### 6.9 Archetype-conditioned EEG topography

问题：

- 当某个群体 `SEEG field-state archetype` 活跃时，EEG 头皮图是否呈现稳定的 archetype-conditioned topography
- 这些 EEG 平均头皮图是否更接近某个 staged EEG microstate template
- 这种关系是单一映射，还是分布式偏好

实现步骤：

1. 复用 `field-state-archetypes` 写出的 subject-to-archetype assignments
2. 在共享 `250 Hz` 时间轴上，把 archetype label 与 staged `19` 导 EEG 样本对齐
3. 对每个病人、每个 archetype 求 archetype-conditioned EEG 平均头皮图
4. 做组水平 scalp map 汇总，并输出 sample-support diagnostics
5. 将 archetype-conditioned scalp maps 与 staged EEG microstate templates 做空间相似性比较
6. 额外输出 `P(EEG state=i | archetype=j)` 一类的零时滞偏好摘要
7. 在近零窗口内继续按原生 `4 ms` 步长评估 archetype 与 EEG microstate 的精细时序同步

解释原则：

- 主结果是 `SEEG archetype-conditioned EEG topography`，不是跨空间直接比较 `Yeo17` 负载和 scalp topography
- 如果空间相似性和零时滞偏好都偏向同一个 EEG microstate，说明存在 archetype-to-microstate preference
- 如果相似性分散或多个 EEG states 都有支持，应保留这种分布式关系，而不是强行一一对应

### 6.10 GFP-informed global coupling

问题：

- EEG 的全局场功率 `GFP` 是否和 SEEG 的全局振幅/能量指标同步或滞后耦合

当前实现会先复用 `run-eeg-states` 阶段写出的 `gfp_trace` 和 `gfp_peaks`，再从 `Yeo17` staged 信号构建 SEEG global metric。预定义指标族如下：

- 主指标：`rms`
  对病人内、网络内标准化后的 `Yeo17 core networks` 做等权 RMS
- 敏感性指标：`envelope-rms`
  对标准化信号做 Hilbert 包络后再取 RMS
- 敏感性指标：`spatial-std`
  对标准化网络信号做跨网络空间离散度汇总
- 可选加权：`sqrt-channel-count`
  用 `sqrt(n_bipolar_channels)` 替代等权

解释顺序是：

1. 先看连续 `EEG GFP <-> SEEG global metric`
2. 再看 `EEG GFP peak` 周围的对称窗口轨迹
3. 最后才看控制 GFP 后，microstate / transition 是否还有额外解释力

### 6.11 GFP peak-centered trajectory

问题：

- 以 EEG GFP peak 为时间零点时，SEEG global metric 在峰前后如何变化

步骤：

1. 读取 staged EEG `gfp_peaks`
2. 在每个 peak 周围取对称窗口 `[-peak_window_sec, +peak_window_sec]`
3. 对每个 offset 汇总 SEEG global metric
4. 通过 circular-shift surrogate 估计结构化 null
5. 在组水平按 offset 聚合效应

注意：

- 这是对称窗口，不预设 EEG 一定领先或滞后于 SEEG

### 6.12 GFP-controlled follow-ups

问题：

- microstate 或 transition 与 SEEG 的关系，是否只是被共享的 GFP 振幅波动驱动

实现分两类：

- `gfp-controlled-microstate`
  在病人内拟合 `SEEG_global(t) ~ GFP(t) + microstate(t)`，再输出按 state 调整后的 SEEG global summary
- `gfp-controlled-transition`
  在病人内围绕 EEG transition 构建窗口，并拟合 `SEEG_global(t) ~ GFP(t) + post_transition_indicator(t)`

解释原则：

- 如果控制 GFP 后效果仍存在，说明状态身份本身提供了额外信息
- 如果控制 GFP 后效果消失，更稳妥的解释是“主要由 shared global drive 驱动”

## 7. 统计策略

当前项目里主要有两种组水平统计：

### 7.1 Four-state omnibus + post-hoc 统计

适用于：

- activity
- 默认 connectivity

方法：

- 先把每个病人的主流程结果整理成四状态 profile
- 对每个 region 或 region pair 做 within-subject four-state omnibus permutation 检验
- 对六个状态对做 pairwise post-hoc 比较
- 默认置换次数：`2048`
- 输出：
  - `n_subjects`
  - `statistic` 或 `mean_effect`
  - `p_perm`
  - `q_fdr`

### 7.2 Exploratory 标量效应统计

适用于：

- event-triggered activity
- windowed coupling
- transition coupling
- event-triggered connectivity

方法：

- 对病人级效应做 sign-flip permutation
- 默认置换次数：`2048`
- exploratory connectivity 额外保留：
  - `mean_state_connectivity`
  - `mean_off_connectivity`

## 8. 输出目录结构

### 8.1 缓存

- `artifacts/cache/index/`
- `artifacts/cache/segments/`
- `artifacts/cache/eeg/`
- `artifacts/cache/seeg/`
- `artifacts/cache/coupling/`
- `artifacts/cache/stats/`

这些目录存放的是可复用缓存，会被后续阶段直接读取。

### 8.2 每次运行目录

- `artifacts/runs/<run-id>/reports/figures/`
- `artifacts/runs/<run-id>/reports/tables/`
- `artifacts/runs/<run-id>/logs/`

其中：

- `reports/` 保存这次命令实际生成的图和表
- `logs/` 保存这次命令的文本日志，记录命令、开始结束时间、config hash 和输出路径

命名规则：

- 文件名包含 `branch`
- 文件名包含运行配置哈希 `runtime_hash`

这意味着：

- 不同参数的结果不会互相覆盖
- exploratory 分支会以 `explore_*` 形式单独存档
- 如果不复用 `--run-id`，一条完整多步工作流通常会对应多个运行目录
- 如果复用同一个 `--run-id`，多步日志和报告会汇总到同一个运行目录
- `reports/figures/` 和 `reports/tables/` 只有在真正写文件时才会被创建
- 当前流程不再预创建空的 `reports/qc/`

## 9. 推荐执行顺序

如果要从零开始完整运行，推荐顺序如下：

```bash
uv run seeg-eegmicrostates build-index
uv run seeg-eegmicrostates run-eeg-states
uv run seeg-eegmicrostates run-seeg-regions
uv run seeg-eegmicrostates run-activity-effects

```

如果你想把这一整轮输出集中到同一个运行目录，给每个命令补同一个 `--run-id` 即可，例如：

```bash
uv run seeg-eegmicrostates build-index --run-id 20260406_230000
uv run seeg-eegmicrostates run-eeg-states --run-id 20260406_230000
uv run seeg-eegmicrostates run-seeg-regions --run-id 20260406_230000
uv run seeg-eegmicrostates run-activity-effects --run-id 20260406_230000

uv run seeg-eegmicrostates run-exploratory-coupling --analysis all --run-id 20260406_230000
uv run seeg-eegmicrostates render-reports --run-id 20260406_230000
```

如果只做公开主流程，前六步中的 exploratory 可以跳过。

## 10. 当前流程的几个重要约束

- EEG 默认依赖固定模板 `ModK.fif`
- SEEG 目前只支持 `AAL3`
- SEEG 只保留严格同区双极道
- exploratory 分析是附加层，不会改变默认主流程结果
- direct EEG-SEEG state coupling 只属于 exploratory，不会变成主流程必经步骤
- GFP-informed global coupling 当前只对 `Yeo17` exploratory 工作流开放

## 11. 一句话总结

当前仓库的核心分析逻辑可以概括为：

“先从所选 IDE 状态时间段内提取 EEG microstate 标签和 AAL3 区域级 SEEG 时序，再从状态条件差异、区域连接变化和探索性时间结构，以及可选的 direct EEG-SEEG state coupling 四个层面评估 EEG 与 SEEG 的耦合关系，并把缓存与每次运行产物分开存放。”
