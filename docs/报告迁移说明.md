# 报告迁移说明

本文档说明 paper-focused workflow 上线后，公开维护的 exploratory surface 和默认报告输出发生了哪些变化。

## 1. 默认报告目录

旧版本默认写：

- `reports/figures/`
- `reports/tables/`

当前默认写：

- `reports/main_tables/`
- `reports/supplementary_tables/`
- `reports/manifests/`

随后由 `scripts/01_main_figures.Rmd` 和 `scripts/02_supplementary_figures.Rmd` 分别写入：

- `reports/main_figures/`
- `reports/supplementary_figures/`

其中 `manifests/` 会记录每个论文图表对应的 `analysis_branch` 和 source cache，方便方法学写作与结果追踪。

补充说明：

- 如果使用 `run-exploratory-coupling --analysis all`，当前应配 `--seeg-parcellation-name yeo17`
- 原因是 `all` 会包含 GFP/global supplementary 分支，而这部分目前只支持 `Yeo17`

## 5. 迁移建议

- 如果你只关心论文主线，优先看 `main_figures/` 与 `main_tables/`。
- 如果你需要机制边界或稳健性结果，再看 `supplementary_figures/` 与 `supplementary_tables/`。
- 如果你要追溯某张图或某个表来自哪个 `explore_*` branch，直接查 `reports/manifests/paper_report_manifest*`。
- 如果你只跑 Python，得到的是可追溯的分类表格；最终论文图由 `scripts/` 下的 R Markdown 工作流生成。
